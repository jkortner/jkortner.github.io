Docker (for Python)

02 September, 2021

I switch between R and Python in my work. The easiest solution to avoid struggles with Python versions between projects is to use Docker (and Docker Desktop). For each project I use a separate image and build the container when I need it.

A very basic set-up is described below. If you want to run an app, you likely have different requirements. For data analysis and development, this basic set-up is a good start.

The project folder contains a Dockerfile, a file with the required Python packages (empty in this example), and a folder with code (hello.py).

project/
 Dockerfile
 requirements.txt
 code/
       hello.py
FROM python:3.8
WORKDIR project
COPY requirements.txt .
RUN pip install -r requirements.txt
CMD [ "bash"]
# hello.py
print("hello")

To build the image, check if it worked, and run the container, use the three lines of code below.

docker build -t img/img .
docker images
docker run --rm -ti -v${PWD}:/project -w/project img/img

Afterwards, you will be in the container and can work as usual with the terminal. To run the script go into the code/ folder and run the file.

/project# cd code
/project/code# python hello.py
>> hello

If you need additional packages, but dont want to re-build the image in a session, you can use pip install.
